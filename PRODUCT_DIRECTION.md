# Product Direction — Policy Pivot

現在正式產品方向：

- 數位繪本不再以 LINE 作為主要閱讀載體。Vercel 上的互動式前端將成為主要閱讀體驗。LINE（如果保留）只做導流入口（短鏈接、按鈕、簡短提示）。

明確規範（不得使用的技術）

- 不使用 Cloudflare Workers 作為正式上線後端。Workers 可保留為實驗或備援用途，但不作為主要服務。 
- 不使用 R2、Cloudflare 物件儲存或任何以 R2 為主的公開 URL（例如 r2.dev）作為正式圖片儲存。 
- 不再以 webhook 關鍵字回應作為核心產品（LINE 不再承擔多頁敘事或複雜互動）。

系統邊界與職責

- 本機後台（作者工具）：負責內容編輯、圖片生成、資料包（靜態 JSON + 圖片）產出。這些工具為作者端用，不部署為閱讀 runtime。 
- 閱讀層（Vercel）：負責承載前端互動閱讀器，讀取作者產出的靜態內容（`public/stories/<story-id>/`），並提供互動、選項與導航。 

部署原則

- 所有閱讀時使用的圖片與故事資料必須以靜態資產形式部署於 Vercel（或等價的靜態主機）。
- 不依賴私有/付費物件儲存作為主要運行時；若有必要使用 CDN，可作為可選優化，但非核心需求。

策略說明

- 產品由「以訊息為中心」轉為「以網頁互動為中心」。此轉向能降低運維複雜度、改善使用者體驗，並讓作者工具與閱讀 runtime有明確分工。
